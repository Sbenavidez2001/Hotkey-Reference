The layers experienced engineers think in
each layer locks in the layers below it — roles before auth, auth before authz, authz before schema, schema before infra
roles → auth → authz → schema → infra
01
Users & Jobs-to-be-Done
Who is this for? Outcomes, not features.
02
Actors & Roles
2–4 distinct user types. Seven = confusing roles with permissions.
03
Domain Entities
The nouns. Account, User, Property. Names only; no schemas yet.
04
Identity (AuthN)
How users prove who they are. SSO, magic links, password.
05
Authorization (AuthZ)
Which roles do what to which entities. Hardest to retrofit.
06
Data Model
Now the tables, columns, indexes. Most decisions already forced.
07
Surfaces & Boundaries
APIs, UI shells, public vs internal.
08
State & Side Effects
Transactional vs eventually consistent.
09
Infra & Deployment
Where code runs. Most teams start here — biggest mistake.
10
Observability
How you'll know it's healthy.
The workflow loop — idea to deployed 70% planning the prompt · 20% testing · 10% build — AI compresses the loop, it doesn't skip it
